# Исправление страницы тестирования Moodle API

## Проблема
Пользователь не может использовать поля для ручного ввода `course_id` и `student_id` на странице `/admin/moodle-test`.

## Внесенные изменения

### 1. Улучшен интерфейс полей для ручного ввода
- Добавлены явные метки для полей ручного ввода
- Улучшена видимость полей
- Добавлен информационный блок с инструкциями

### 2. Исправлена логика JavaScript
- Улучшена обработка значений из select и manual input
- Добавлена проверка доступности полей при загрузке страницы
- Убраны атрибуты `disabled` и `readonly` при необходимости

### 3. Добавлены инструкции
- Информационный блок в начале формы объясняет, как использовать поля
- Улучшены подсказки под полями

## Как использовать

### Для ID курса:
1. **Вариант 1:** Выберите курс из выпадающего списка
2. **Вариант 2:** Введите ID курса в Moodle вручную в поле "Или введите ID курса в Moodle вручную"

### Для ID студента:
1. **Вариант 1:** Выберите студента из выпадающего списка
2. **Вариант 2:** Введите ID студента в системе вручную (не Moodle ID!) в поле "Или введите ID студента вручную"

## Важные замечания

1. **Авторизация:** Убедитесь, что вы авторизованы в системе и имеете роль `instructor` или `admin`
2. **ID студента:** При ручном вводе используйте ID студента в системе (из таблицы `users`), а не Moodle User ID
3. **ID курса:** При ручном вводе используйте ID курса в Moodle (Moodle Course ID)

## Проверка работы

1. Откройте страницу `/admin/moodle-test`
2. Убедитесь, что вы видите информационный блок с инструкциями
3. Попробуйте ввести ID курса вручную в поле "Или введите ID курса в Moodle вручную"
4. Попробуйте ввести ID студента вручную в поле "Или введите ID студента вручную"
5. Нажмите "Выполнить тест"

## Если поля все еще не работают

1. Откройте консоль браузера (F12) и проверьте наличие ошибок JavaScript
2. Убедитесь, что вы авторизованы (проверьте, что не перенаправляет на страницу входа)
3. Проверьте, что у вас есть роль `instructor` или `admin`
4. Очистите кеш браузера и попробуйте снова
